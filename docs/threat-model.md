# Модель угроз (Secure Notes API)

## Контур
- API endpoint'ы: auth + notes CRUD
- Данные: учетные данные пользователей, JWT, содержимое заметок
- Инфраструктура: runtime в Docker Compose, проверки в CI-пайплайне

## Активы
- Аккаунты пользователей и хеши паролей
- Секрет подписи JWT
- Данные заметок
- CI-артефакты и манифесты зависимостей

## Точки входа
- `/auth/register`
- `/auth/login`
- `/notes*`
- Контейнерный образ и CI-пайплайн

## Основные угрозы
- Injection-атаки на запросы заметок и auth-flows
- Credential stuffing / brute-force против логина
- Утечки секретов (hardcoded keys, открытые env vars)
- Уязвимые зависимости и базовые образы
- Ошибки конфигурации в деплое и IaC

## Реализованные security-контроли
- Хеширование паролей (`bcrypt`)
- Проверка подписи JWT и срока действия токена
- SQLAlchemy ORM с параметризованными запросами по умолчанию
- Валидация входных данных через схемы Pydantic
- Изоляция AuthZ: пользователи имеют доступ только к своим заметкам
- Многоуровневое сканирование в CI (SAST, SCA, container scan, DAST)

## Остаточные риски
- Нет MFA
- В этой starter-версии нет WAF/rate limiting
- Нет централизованной интеграции с SIEM
